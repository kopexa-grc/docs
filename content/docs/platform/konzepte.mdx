Kopexa organisiert Compliance-Arbeit in zwei Ebenen: **Organisation** (Mandant) und **Spaces** (Arbeitsbereiche).

## Organisation

Die Organisation ist dein Top-Level-Container.

| Funktion | Beschreibung |
|----------|--------------|
| Domains & SSO | SAML-Provider einrichten, Domain verifizieren |
| Nutzer | Einladen, Spaces zuweisen |
| Abrechnung | Plan, Zahlungsmethode |
| Einstellungen | Globale Security-Policies |

→ [Organisation einrichten](./quickstart/organization/domains/index.mdx)

## Spaces

Spaces sind isolierte Arbeitsbereiche innerhalb der Organisation.

- Ein Space pro Team, Projekt oder Abteilung
- Daten zwischen Spaces sind strikt getrennt
- Jeder Space hat eigene Module

**Beispiel:** Hauptunternehmen = Space 1, Tochtergesellschaft = Space 2.

→ [Spaces](./quickstart/spaces.mdx)

## Module

Innerhalb eines Spaces:

| Modul | Funktion |
|-------|----------|
| **Inventory** | Assets, Lieferanten, Personen |
| **Risiken** | Risikobewertung, Maßnahmen |
| **Kontrollen** | Framework-Anforderungen |
| **Dokumente** | Policies, Prozesse |
| **Nachweis** | Nachweise |
| **Aufgaben** | Aufgaben |
| **VVT** | Verarbeitungstätigkeiten (DSGVO) |
| **Vorfälle** | Sicherheitsvorfälle |
| **DSAR** | Betroffenenanfragen |

## Beziehungen
