title: Quickstart
description: In 15 Minuten zum ersten dokumentierten Risiko mit Kontrolle und Maßnahme
---

Diese Anleitung führt dich durch die ersten Schritte in Kopexa. Am Ende hast du:

- eine Organisation mit verifizierter Domain
- einen Space für dein Team
- ein aktiviertes Framework (ISO 27001)
- ein dokumentiertes Risiko
- eine zugewiesene Kontrolle
- eine angelegte Maßnahme
- einen ersten Nachweis

## Voraussetzungen

- Ein Kopexa-Account (Login unter [app.kopexa.com](https://app.kopexa.com/auth/login))
- Zugriff auf die DNS-Einstellungen deiner Domain (für die Verifizierung)

## 1. Organisation erstellen

Nach dem ersten Login startet der Einrichtungs-Assistent. Trage die Eckdaten deiner Organisation ein:

| Feld | Beispiel |
|------|----------|
| Name | Mustermann GmbH |
| Slug | mustermann-gmbh |
| Branche | IT-Dienstleistungen |

Der Slug wird Teil deiner URL: `app.kopexa.com/o/mustermann-gmbh`.

Klicke auf **Organisation erstellen**.

## 2. Domain verifizieren

Damit dein Team sich per Single Sign-On anmelden kann, verifiziere zunächst die Domain.

<Steps>
<Step>Öffne **Einstellungen → Domains**.</Step>
<Step>Klicke auf **Domain hinzufügen**.</Step>
<Step>Trage deine Domain ein, zum Beispiel `mustermann-gmbh.de`.</Step>
<Step>Kopexa zeigt dir einen DNS-Eintrag wie diesen:

```
_kopexa-verification.mustermann-gmbh.de  TXT  "kopexa-verify=abc123xyz"
```
</Step>
<Step>Trage diesen Eintrag bei deinem DNS-Provider ein (zum Beispiel Cloudflare, AWS Route 53).</Step>
<Step>Warte ein bis fünf Minuten und klicke auf **Verifizieren**.</Step>
</Steps>

<Callout type="info">
Die weltweite Verteilung von DNS-Einträgen kann bis zu 24 Stunden dauern. In der Praxis sind es meist wenige Minuten.
</Callout>

## 3. Ersten Space anlegen

Ein Space ist ein abgegrenzter Arbeitsbereich. Für den Einstieg reicht einer.

1. Klicke auf **Spaces → Neuer Space**.
2. Trage Name und Beschreibung ein:

| Feld | Beispiel |
|------|----------|
| Name | Hauptunternehmen |
| Beschreibung | ISMS für die Mustermann GmbH |

3. Klicke auf **Space erstellen**. Du wirst direkt in den neuen Space weitergeleitet.

## 4. Framework aktivieren

Jetzt aktivierst du ISO 27001 als Grundlage für deine Kontrollen.

1. Im Space: **Frameworks** öffnen.
2. **ISO 27001** in der Liste finden.
3. Auf **Aktivieren** klicken.

Im Hintergrund werden alle zugehörigen Kontrollen aus dem Katalog übernommen. Das dauert nur wenige Sekunden.

<Callout type="info">
Du kannst später weitere Frameworks aktivieren, etwa NIS2, DSGVO oder TISAX®. Überschneidende Anforderungen werden automatisch verknüpft.
</Callout>

## 5. Erstes Asset anlegen

Bevor du Risiken bewertest, brauchst du etwas Konkretes, auf das sie sich beziehen. Lege als Beispiel ein Asset an.

1. Öffne **Assets** im Bereich Organisation.
2. Klicke auf **Neues Asset**.
3. Trage ein:

| Feld | Beispiel |
|------|----------|
| Name | Produktionsserver |
| Typ | Server |
| Beschreibung | Linux-Server für Kundenprojekte |
| Owner | IT-Abteilung |
| Kritikalität | Hoch |

4. Klicke auf **Erstellen**.

## 6. Erstes Risiko dokumentieren

Jetzt dokumentierst du ein Risiko für dieses Asset.

1. Öffne **Risiken**.
2. Klicke auf **Neues Risiko**.
3. Trage ein:

| Feld | Wert |
|------|------|
| Titel | Unautorisierter Zugriff auf Produktionsserver |
| Beschreibung | Angreifende könnten durch gestohlene Zugangsdaten oder ungepatchte Lücken Zugang erlangen |
| Asset | Produktionsserver |
| Kategorie | Vertraulichkeit |

4. Bewerte das Risiko:

| Dimension | Wert | Begründung |
|-----------|------|------------|
| Auswirkung | Hoch (4) | Kundendaten betroffen, Reputationsschaden |
| Eintrittswahrscheinlichkeit | Mittel (3) | Server ist exponiert, aber grundlegend gehärtet |

Die Risikomatrix zeigt: Risikowert 12 (Hoch).

5. Klicke auf **Erstellen**.

## 7. Kontrolle zuweisen

Verknüpfe das Risiko mit einer passenden Kontrolle aus ISO 27001.

1. Öffne das eben erstellte Risiko.
2. Wechsle in den Bereich **Kontrollen**.
3. Klicke auf **Kontrolle verknüpfen**.
4. Suche nach "Zugang" oder "Access".
5. Wähle zum Beispiel **A.5.15 Access Control**.
6. Bestätige die Verknüpfung.

Die Kontrolle beschreibt das *Was*: die Anforderung aus dem Framework.

## 8. Maßnahme anlegen

Eine Maßnahme beschreibt, *wie* du die Kontrolle in deiner Organisation umsetzt.

1. Öffne die verknüpfte Kontrolle.
2. Wechsle in den Bereich **Maßnahmen**.
3. Klicke auf **Neue Maßnahme**.
4. Trage ein:

| Feld | Wert |
|------|------|
| Titel | Zentrale Benutzerverwaltung über Active Directory |
| Beschreibung | Alle Benutzerkonten werden zentral über AD gepflegt. Onboarding und Offboarding laufen über das Ticket-System mit Vier-Augen-Prinzip. |
| Status | Teilweise umgesetzt |
| Owner | Max Mustermann |
| Fälligkeit | In 30 Tagen |

5. Klicke auf **Erstellen**.

<Callout type="info">
**Kontrolle gegen Maßnahme:** Die Kontrolle sagt "Benutzerregistrierung muss gesteuert ablaufen" (Was). Die Maßnahme sagt "Wir nutzen AD mit Ticket-basiertem Onboarding" (Wie).
</Callout>

## 9. Nachweis hochladen

Dokumentiere abschließend, dass die Maßnahme umgesetzt ist.

1. In der Maßnahme: Bereich **Nachweise**.
2. Klicke auf **Nachweis hinzufügen**.
3. Wähle eine Datei aus (zum Beispiel einen Screenshot deiner AD-Konsole oder ein Onboarding-Ticket).
4. Beschreibung: "AD-Benutzerverwaltung, Stand Q1 2026".
5. Klicke auf **Hochladen**.

## Geschafft

Du hast jetzt:

- eine Organisation mit verifizierter Domain
- einen Space
- ISO 27001 aktiviert
- ein Asset dokumentiert
- ein Risiko bewertet
- eine Kontrolle zugewiesen
- eine Maßnahme angelegt
- einen Nachweis hochgeladen

Das ist der Kern-Ablauf in Kopexa:

**Asset → Risiko → Kontrolle → Maßnahme → Nachweis**

- **Kontrolle**: Was muss erfüllt werden (Framework-Anforderung)
- **Maßnahme**: Wie setzen wir es um (konkrete Umsetzung)
